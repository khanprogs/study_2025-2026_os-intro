Задание 5 — Краткая характеристика команд
mount — монтирует файловую систему, то есть подключает устройство к дереву каталогов. Без параметров выводит список всех смонтированных файловых систем. Пример: mount /dev/sdb1 /mnt/usb монтирует раздел /dev/sdb1 в точку /mnt/usb.
fsck (file system check) — проверяет целостность файловой системы и при возможности устраняет обнаруженные ошибки. Рекомендуется запускать на отмонтированном разделе. Пример: fsck /dev/sda1.
mkfs (make file system) — создаёт новую файловую систему на указанном устройстве или разделе. Пример: mkfs -t ext4 /dev/sdb1 создаёт файловую систему ext4 на разделе /dev/sdb1.
kill — отправляет сигнал процессу по его PID. По умолчанию отправляется сигнал SIGTERM (завершение). Пример: kill -9 1234 принудительно завершает процесс с PID 1234 сигналом SIGKILL.

# Ответы на контрольные вопросы

1. Характеристика файловых систем на жёстком диске
На диске, как правило, используются ext4 (основной раздел с операционной системой) и swap (раздел подкачки). ext4 — журналируемая файловая система с поддержкой больших файлов и томов, высокой надёжностью и производительностью. swap не является файловой системой в классическом смысле: это раздел подкачки, используемый ядром при нехватке оперативной памяти.
2. Общая структура файловой системы Linux
Файловая система Linux имеет иерархическую структуру с единым корнем /. Основные каталоги первого уровня: /bin — базовые исполняемые файлы; /etc — конфигурационные файлы системы; /home — домашние каталоги пользователей; /var — изменяемые данные (логи, кэш); /tmp — временные файлы; /usr — пользовательские программы и библиотеки; /dev — файлы устройств; /proc — виртуальная файловая система с информацией о процессах и ядре; /root — домашний каталог администратора; /lib — библиотеки, необходимые для загрузки системы.
3. Монтирование файловой системы
Чтобы содержимое файловой системы стало доступным, её необходимо смонтировать командой mount, указав устройство и точку монтирования. Монтирование связывает файловую систему устройства с определённым каталогом в дереве каталогов Linux.
4. Причины нарушения целостности и её восстановление
Основные причины: некорректное выключение системы, сбои питания, аппаратные ошибки дискового контроллера, программные ошибки. Для проверки и восстановления целостности используется утилита fsck, которую следует запускать на отмонтированном разделе.
5. Создание файловой системы
Файловая система создаётся утилитой mkfs. Например, mkfs -t ext4 /dev/sdb1 форматирует раздел /dev/sdb1 в ext4. Перед созданием файловой системы на разделе все хранящиеся на нём данные уничтожаются.
6. Команды для просмотра текстовых файлов
cat — выводит весь файл целиком, удобен для небольших файлов. less — постраничный просмотр с возможностью навигации вперёд и назад. head — выводит первые строки файла (по умолчанию 10). tail — выводит последние строки файла (по умолчанию 10).
7. Основные возможности команды cp
Копирование одного файла в другой; копирование нескольких файлов в каталог; рекурсивное копирование каталогов с флагом -r; запрос подтверждения перезаписи с флагом -i; сохранение атрибутов исходного файла с флагом -p.
8. Основные возможности команды mv
Переименование файлов и каталогов; перемещение файлов и каталогов в другое место файловой системы; запрос подтверждения перезаписи с флагом -i. В отличие от cp, исходный файл после перемещения удаляется.
9. Права доступа и их изменение
Права доступа определяют, какие операции (чтение r, запись w, выполнение x) могут выполнять с файлом или каталогом его владелец, члены группы и остальные пользователи. Изменяются командой chmod. Права можно задавать в символьном виде (chmod u+x file) или в восьмеричном (chmod 755 file).
